# Deployment Guide

This guide covers deploying the portfolio application to Azure Container Apps (ACA).

## Architecture

```
Internet → DNS CNAME → ACA Ingress (HTTPS, managed TLS)
                        ↓
             Azure Container Apps Environment (VNet)
                        ↓
             portfolio-app (Next.js container from ACR, Port 3000)
              /                          \
PostgreSQL Flexible Server          Azure Blob Storage
(private, VNet-only)               (public, portfolio-images)

ACR (portfolioacr2026) ← GitHub Actions (OIDC auth, no secrets in code)
```

| Component | Purpose |
|-----------|---------|
| **Azure Container Apps** | Hosts the Next.js application as a Docker container |
| **Azure Container Registry** | Private Docker image storage |
| **PostgreSQL Flexible Server** | Production database (VNet-private, no public access) |
| **Azure Blob Storage** | Image hosting (`portfoliodevstore2026`) |
| **GitHub Actions** | CI/CD pipeline with OIDC authentication |
| **Google OAuth** | Admin authentication via email allowlist |

### Why This Architecture

| Decision | Reasoning |
|----------|-----------|
| **ACA over App Service** | Container-based deployment avoids Oryx build issues. ACA runs your Docker image exactly as built. |
| **ACR** | Private container registry in the resource group. ACA pulls directly. |
| **PostgreSQL in VNet** | Database has no public endpoint. Only ACA containers can reach it. |
| **Blob Storage stays public** | Images must be browser-accessible. Already working. |
| **ACA ingress (not Front Door)** | Built-in HTTPS + managed TLS is sufficient for portfolio traffic. Front Door can be added later without changes. |
| **GitHub Actions OIDC** | No stored secrets. Short-lived tokens replace permanent credentials. |

## Dockerfile

The application uses a multi-stage Docker build for a minimal production image. Create `Dockerfile` in the project root:

```dockerfile
# Stage 1: Install dependencies
FROM node:22-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --ignore-scripts
# Prisma needs to generate after install
COPY src/prisma ./src/prisma
COPY prisma.config.ts ./
RUN npx prisma generate

# Stage 2: Build the application
FROM node:22-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Stage 3: Production image
FROM node:22-alpine AS runner
WORKDIR /app

ENV NODE_ENV=production
ENV PORT=3000

RUN addgroup --system --gid 1001 nodejs && \
    adduser --system --uid 1001 nextjs

# Copy standalone output
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

# Copy Prisma schema + generated client for runtime queries
COPY --from=builder /app/src/prisma ./src/prisma
COPY --from=builder /app/prisma.config.ts ./
COPY --from=builder /app/node_modules/.prisma ./node_modules/.prisma
COPY --from=builder /app/node_modules/@prisma ./node_modules/@prisma

USER nextjs
EXPOSE 3000

CMD ["node", "server.js"]
```

Also create `.dockerignore`:

```
node_modules
.next
.env*
ai_working
test-assets
*.md
.git
```

### Notes

- **`server.js`** is generated by Next.js standalone output (`next build` creates `.next/standalone/server.js`). It does not need to exist in the repo.
- **Prisma migrations run in CI/CD**, not on container start. This avoids migration races if multiple containers start simultaneously.
- **`sharp`** works with `node:22-alpine`. If image processing issues arise, add `RUN apk add --no-cache vips-dev` to the runner stage.

### Local Docker Testing

Build and test the container locally before deploying to Azure:

```bash
# Build the image
docker build -t portfolio-app .

# Verify image size (~200-300MB is expected)
docker images portfolio-app

# Run the container (Docker Compose must be up for PostgreSQL)
docker run --rm -p 3000:3000 \
  -e DATABASE_URL="postgresql://postgres:postgres@host.docker.internal:5432/portfolio" \
  -e AUTH_SECRET="<your-dev-secret>" \
  -e GOOGLE_CLIENT_ID="<your-id>" \
  -e GOOGLE_CLIENT_SECRET="<your-secret>" \
  -e NEXTAUTH_URL="http://localhost:3000" \
  portfolio-app
```

**Why `host.docker.internal`?** A Docker container has its own network namespace. `localhost` inside the container refers to the container itself, not your Mac. `host.docker.internal` is Docker Desktop's special DNS name that resolves to your host machine's IP.

**Test checklist:**
1. Public pages load (homepage, categories, projects)
2. Google OAuth login works (complete the full redirect flow)
3. Admin pages load after login
4. Create or edit content — verify it persists
5. Images display correctly

## Azure Infrastructure Setup

All resources go in `portfolio-rg` (westus2).

### Resources to Create

| Resource | Name | SKU/Tier |
|----------|------|----------|
| Virtual Network | `portfolio-vnet` | - |
| Subnet (ACA) | `aca-infra` (10.0.0.0/23) | - |
| Subnet (PostgreSQL) | `postgres-subnet` (10.0.2.0/24) | - |
| PostgreSQL Flexible Server | `portfolio-db` | Burstable B1ms |
| Container Registry | `portfolioacr2026` | Basic |
| Container Apps Environment | `portfolio-env` | Consumption |
| Container App | `portfolio-app` | 0.25 vCPU, 0.5GB |
| Entra ID App Registration | `portfolio-github-deploy` | - |

### Resources That Already Exist

| Resource | Name | Notes |
|----------|------|-------|
| Resource Group | `portfolio-rg` | westus2 |
| Storage Account | `portfoliodevstore2026` | Blob storage for images. No changes needed. |

### Step 1: Networking

```bash
# Create VNet
az network vnet create \
  --resource-group portfolio-rg \
  --name portfolio-vnet \
  --location westus2 \
  --address-prefixes 10.0.0.0/16

# Create ACA subnet (requires minimum /23)
az network vnet subnet create \
  --resource-group portfolio-rg \
  --vnet-name portfolio-vnet \
  --name aca-infra \
  --address-prefixes 10.0.0.0/23

# Create PostgreSQL subnet with delegation
az network vnet subnet create \
  --resource-group portfolio-rg \
  --vnet-name portfolio-vnet \
  --name postgres-subnet \
  --address-prefixes 10.0.2.0/24 \
  --delegations Microsoft.DBforPostgreSQL/flexibleServers
```

### Step 2: PostgreSQL

Create the database server with **public access initially** so you can run migrations from your laptop. It gets locked down to VNet-only after the app is verified.

```bash
az postgres flexible-server create \
  --resource-group portfolio-rg \
  --name portfolio-db \
  --location westus2 \
  --admin-user portfolioadmin \
  --admin-password "<strong-password>" \
  --sku-name Standard_B1ms \
  --tier Burstable \
  --storage-size 32 \
  --version 16 \
  --public-access 0.0.0.0

az postgres flexible-server db create \
  --resource-group portfolio-rg \
  --server-name portfolio-db \
  --database-name portfolio
```

### Step 3: Container Registry

```bash
az acr create \
  --resource-group portfolio-rg \
  --name portfolioacr2026 \
  --sku Basic
```

### Step 4: Container Apps Environment

```bash
az containerapp env create \
  --name portfolio-env \
  --resource-group portfolio-rg \
  --location westus2 \
  --infrastructure-subnet-resource-id \
    "/subscriptions/<sub-id>/resourceGroups/portfolio-rg/providers/Microsoft.Network/virtualNetworks/portfolio-vnet/subnets/aca-infra"
```

### Step 5: Bootstrap the Database

Run these from your local machine while PostgreSQL has public access:

```bash
# Store the connection string for this session
export AZURE_DB_URL="postgresql://portfolioadmin:<password>@portfolio-db.postgres.database.azure.com:5432/portfolio?sslmode=require"

# Apply schema (creates all tables)
DATABASE_URL="$AZURE_DB_URL" npx prisma migrate deploy

# Verify tables were created
psql "$AZURE_DB_URL" -c "\dt"

# Seed your admin email
DATABASE_URL="$AZURE_DB_URL" npx tsx src/prisma/seed-admin.ts your-email@gmail.com
```

### Step 6: Build and Deploy the App

```bash
# Build image in ACR
az acr build --registry portfolioacr2026 --image portfolio-app:v1 .

# Create the container app with all env vars
az containerapp create \
  --name portfolio-app \
  --resource-group portfolio-rg \
  --environment portfolio-env \
  --image portfolioacr2026.azurecr.io/portfolio-app:v1 \
  --target-port 3000 \
  --ingress external \
  --min-replicas 0 \
  --max-replicas 2 \
  --registry-server portfolioacr2026.azurecr.io \
  --env-vars \
    DATABASE_URL="postgresql://portfolioadmin:<password>@portfolio-db.postgres.database.azure.com:5432/portfolio?sslmode=require" \
    GOOGLE_CLIENT_ID="<your-google-client-id>" \
    GOOGLE_CLIENT_SECRET="<your-google-client-secret>" \
    AUTH_SECRET="$(openssl rand -base64 32)" \
    NEXTAUTH_URL="https://<aca-default-fqdn>" \
    AZURE_STORAGE_CONNECTION_STRING="<storage-connection-string>" \
    AZURE_STORAGE_CONTAINER_NAME="portfolio-images" \
    NODE_ENV="production"
```

### Step 7: Populate Data and Lock Down

```bash
# Populate persona data against the running app
API_BASE="https://<aca-fqdn>/api" node scripts/populate-persona-api.js sarah-chen

# Once verified, lock down PostgreSQL to VNet-only
az postgres flexible-server update \
  --name portfolio-db \
  --resource-group portfolio-rg \
  --public-access Disabled
```

## Environment Variables

### Required for Production

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string (must include `?sslmode=require`) |
| `GOOGLE_CLIENT_ID` | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret |
| `AUTH_SECRET` | Session encryption key (generate with `openssl rand -base64 32`) |
| `NEXTAUTH_URL` | Full URL of your site (`https://portfolio.sashagoodner.com`) |
| `AZURE_STORAGE_CONNECTION_STRING` | Azure Blob Storage connection string |
| `AZURE_STORAGE_CONTAINER_NAME` | Blob container name (`portfolio-images`) |
| `NODE_ENV` | `production` |

### How Environments Differ

| Signal | Local | Production |
|--------|-------|------------|
| `NODE_ENV` | `development` | `production` |
| Database | `localhost:5432` | `portfolio-db.postgres.database.azure.com` |
| Image storage | Files in `public/uploads/` | Blobs at `portfoliodevstore2026.blob.core.windows.net` |
| URL | `http://localhost:3000` | `https://portfolio.sashagoodner.com` |

## CI/CD Pipeline

The pipeline uses GitHub Actions with OIDC authentication (no stored secrets).

### Setup OIDC

```bash
# Create Entra ID App Registration
az ad app create --display-name "portfolio-github-deploy"

# Note the appId, then create service principal
az ad sp create --id <appId>

# Assign roles
az role assignment create \
  --assignee <appId> \
  --role Contributor \
  --scope /subscriptions/<sub-id>/resourceGroups/portfolio-rg

az role assignment create \
  --assignee <appId> \
  --role AcrPush \
  --scope /subscriptions/<sub-id>/resourceGroups/portfolio-rg

# Add federated credential for GitHub
az ad app federated-credential create --id <appId> --parameters '{
  "name": "github-main",
  "issuer": "https://token.actions.githubusercontent.com",
  "subject": "repo:<github-user>/portfolio:ref:refs/heads/main",
  "audiences": ["api://AzureADTokenExchange"]
}'
```

### GitHub Secrets & Variables

**Secrets** (encrypted, masked in logs):

| Secret | Contains |
|--------|----------|
| `AZURE_CLIENT_ID` | Entra app registration client ID |
| `AZURE_TENANT_ID` | Entra tenant ID |
| `AZURE_SUBSCRIPTION_ID` | Azure subscription ID |
| `ACR_LOGIN_SERVER` | `portfolioacr2026.azurecr.io` |
| `DATABASE_URL` | Full PostgreSQL connection string |
| `GOOGLE_CLIENT_ID` | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret |
| `AUTH_SECRET` | NextAuth session encryption key |
| `AZURE_STORAGE_CONNECTION_STRING` | Blob storage connection string |

**Variables** (non-sensitive, hidden from code):

| Variable | Contains |
|----------|----------|
| `RESOURCE_GROUP` | `portfolio-rg` |
| `ACA_NAME` | `portfolio-app` |
| `ACR_NAME` | `portfolioacr2026` |
| `IMAGE_NAME` | `portfolio-app` |
| `ACA_ENVIRONMENT` | `portfolio-env` |
| `STORAGE_CONTAINER` | `portfolio-images` |
| `SITE_URL` | `https://portfolio.sashagoodner.com` |

### Workflow

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches: [main]

permissions:
  id-token: write   # Required for OIDC
  contents: read

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Azure Login (OIDC)
        uses: azure/login@v2
        with:
          client-id: ${{ secrets.AZURE_CLIENT_ID }}
          tenant-id: ${{ secrets.AZURE_TENANT_ID }}
          subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}

      - name: ACR Login
        run: az acr login --name ${{ vars.ACR_NAME }}

      - name: Build and Push Image
        run: |
          docker build \
            -t ${{ secrets.ACR_LOGIN_SERVER }}/${{ vars.IMAGE_NAME }}:${{ github.sha }} \
            -t ${{ secrets.ACR_LOGIN_SERVER }}/${{ vars.IMAGE_NAME }}:latest \
            .
          docker push ${{ secrets.ACR_LOGIN_SERVER }}/${{ vars.IMAGE_NAME }} --all-tags

      - name: Setup Node (for Prisma CLI)
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Run Database Migrations
        run: |
          npm ci --ignore-scripts
          npx prisma migrate deploy
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}

      - name: Deploy to Container Apps
        run: |
          az containerapp update \
            --name ${{ vars.ACA_NAME }} \
            --resource-group ${{ vars.RESOURCE_GROUP }} \
            --image ${{ secrets.ACR_LOGIN_SERVER }}/${{ vars.IMAGE_NAME }}:${{ github.sha }}
```

## Custom Domain & SSL

See [Domain Setup Guide](./DOMAIN_SETUP.md) for detailed instructions.

Quick summary:

```bash
# Add custom hostname to ACA
az containerapp hostname add \
  --name portfolio-app \
  --resource-group portfolio-rg \
  --hostname portfolio.sashagoodner.com

# Bind managed certificate (free, auto-renewing)
az containerapp hostname bind \
  --name portfolio-app \
  --resource-group portfolio-rg \
  --hostname portfolio.sashagoodner.com \
  --environment portfolio-env \
  --validation-method CNAME
```

Then configure DNS at your registrar:
```
CNAME  portfolio  →  <aca-default-fqdn>.azurecontainerapps.io
```

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| 500 error on startup | Missing environment variables | Check all required vars are set on ACA |
| Database connection fails | SSL or credentials | Verify `?sslmode=require` in DATABASE_URL |
| OAuth "Bad Request" | Wrong NEXTAUTH_URL | Verify it matches the site's actual URL |
| Images not uploading | Storage not configured | Check AZURE_STORAGE_* variables |
| Container won't start | Build error in image | Test locally with `docker build` + `docker run` first |
| Cold start slow (5-10s) | ACA scale-to-zero | Normal for Consumption plan. First request spins up container. |
